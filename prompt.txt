

## **Prompt for THINKING — Full Implementation of Drone-based Signal Scanning System Simulator & Algorithms**

We require a **complete, modular Python implementation** of our autonomous drone-based scanning system for locating missing persons via WiFi and BLE signal detection. The deliverable must include:

---

### **1. System Overview**

The simulation must model the **real-world architecture** from our project spec:

* **Environment**: 3D space (100 m × 100 m × 20 m).
* **Emitter(s)**: Representing phones or BLE devices with fixed random position, realistic Tx power, Gaussian white noise, and chosen protocol (WiFi or BLE). Packets must mimic realistic probe requests (WiFi) or advertisements (BLE).
* **Scanner(s)**: Representing the drone’s scanning unit, starting at a random position, moving per a configurable flight pattern (lawnmower, random walk, or waypoint path), collecting RSSI measurements with position and timestamp.
* **Output**: CSV file with all scan data:
  `ts, x, y, z, rssi, mac_hash, proto, scanner_id, emitter_id(optional)`

---

### **2. Simulator Architecture**

**Modules**:

```
core/         # Environment, timebase, entity base classes
protocols/    # WiFi, BLE packet simulators
channel/      # Path-loss, AWGN, shadowing, multipath models
actors/       # Emitter, Scanner, Motion models
algos/        # Multilateration, KF/EKF, Particle Filter, Gaussian Process
io/           # CSV writer, config loader
visualize/    # 2D/3D plots, heatmaps
cli/          # Run simulation, run post-processing
tests/        # Unit & E2E tests
```

**Modularity**:

* Each algorithmic block is replaceable via config.
* Future support for multiple drones and emitters without major refactoring.
* Configurable parameters in YAML/TOML.

---

### **3. Signal & Channel Model**

* **Path-loss model**: Log-distance with parameters `pl0_db`, `n`, `d0_m`.
* **Noise**: Gaussian white noise (σ configurable).
* **Shadowing**: Log-normal shadowing (σ configurable).
* **RSSI inversion**:

  $$
  \hat d = d_0 \cdot 10^{\frac{P_{\text{tx}} - \text{PL}_0 - \text{RSSI}}{10\,n}}
  $$

  Clamp to `[min_d_m, max_d_m]`.

---

### **4. Flight & Motion Models**

Implement:

* **Lawnmower (grid)** pattern (default for initial scan).
* **Random walk**.
* **Waypoint path**.
  Configurable speed, altitude, spacing.

---

### **5. Online Localization Algorithm**

Processing chain for real-time estimation:

1. **RSSI→Distance** using path-loss inversion.
2. **Multilateration**:

   * Weighted Least Squares (Huber or soft-L1 robust loss).
   * Output: `(x,y,z)`, covariance, residual RMS.
3. **KF/EKF Smoother**:

   * State: `[x,y,z,vx,vy,vz]`, constant velocity model.
   * Tunable `Q` and `R` matrices.
4. **Particle Filter / Grid Heatmap**:

   * Grid or particle representation of position probability.
   * Update likelihood from RSSI model.
   * Maintain normalized weights and produce a real-time heatmap.

---

### **6. Offline Post-Processing**

* **Gaussian Process Regression (GPR)**: Fit `f(x,y)=RSSI` over scanned area, output smooth heatmap and uncertainty map.
* Identify peak RSSI locations for final refined position.
* Save:

  * PNG heatmap (mean)
  * PNG uncertainty map (std)
  * JSON with peak coordinates and confidence score.

Example YAML:

```yaml
  awgn_sigma_db: 1.0

  pf: {mode: "grid", grid_res: 2.0, N: 8000}
  gp: {length_scale: 10.0, noise: 2.0}
io:
  out_csv: "runs/run001.csv"
  out_dir: "runs/"
```
